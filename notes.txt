        * Generate a probability that the opponent is holding a given card
            for each unknown card. Then run ISMCTS or PIMC using these

            just be an even distribution. If we use PIMC, we can
            use Double Dummy to completely solve a given world

            correcet probabilities of unknown cards.

            NN-assisted determinization was so lackluster, I'm not

            train a REINFORCE only model (no search, no MCTS) for

* Approaches tried
    - ISMCTS + NN
        * The idea was to match chess-ai-engine's (https://github.com/johnnyodonnell/chess-ai-engine)
            AlphaZero approach, but we just didn't get very good results
            from it. The conclusion was that this is not a good approach
            for this game. And if it is in fact a good approach, we're
            certainly not implementing it correctly.
        * We were finding that ISMCTS wasn't adding much value on top
            of our already trained REINFORCE model. It helped a little
            but not much. Then, if we trained a model using ISMCTS, that
            model w/o ISMCTS would actually perform better against
            itself w/ ISMCTS.
        * The hardest part about this game is determining where the unknown
            cards are. Once you can do that, knowing what card to play next
            is very straightforward, which is different from chess.
            So the gains that we saw from using search in chess just may
            not apply here.
    - AWR
        * Did not have much success here
        * This was suggested early in our project, which is why it was tried
        * It was also appealing because the model could be trained off-policy,
            but we just didn't get any good results from this approach
        * After some research, we couldn't find any examples that used this
            approach for training self-play game engines. The examples
            we found were mainly used for robotics.

* Why ISMCTS search may not work for this game
    - The flywheel only spins if search beats the raw net. The whole premise of
        AlphaZero-style training is a loop: the search uses the net to produce moves
        better than the net's raw output, those better moves generate better games,
        those games become training targets, the net improves, repeat. Search is the
        amplifier in that loop.
    - But our A/B showed the amplifier is currently doing nothing: run3 with search
        scored 44/100 against its own raw policy. Search isn't improving on the net,
        so the self-play data is no better than what the policy alone would produce,
        and training crawls.
    - That's the "value hostage" dynamic: the net can only get better through search,
        and search can only help if the net is already decent.
